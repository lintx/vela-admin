# 长期计划

本目录用于存放需要加入 git 跟踪的长期计划。长期计划回答“接下来分阶段做什么、执行到哪里”，不同于 `docs/development/` 中稳定的维护规则，也不同于 `docs/adr/` 中记录的决策原因。

## 存放范围

适合放入本目录：

1. 规范治理计划。
2. 跨多个任务的重构、迁移或发布计划。
3. 需要阶段性更新状态的路线执行计划。
4. 后续项目 skill 抽取计划。

不适合放入本目录：

1. 当前一次性任务草稿。
2. 临时排查步骤。
3. 执行完即失效的短期计划。

短期计划应放入 `.task-plans/`，不纳入 git 跟踪，任务结束后删除。

## 完成后处理

长期计划完成后必须收尾，不应永久堆在本目录中：

1. 如果计划只有过程价值，应先把最终结论合并到 `docs/development/`、`docs/adr/`、`docs/roadmap.md` 或其他稳定文档，再删除计划文件。
2. 如果计划本身有长期查阅价值，可以保留在本目录，并在文件顶部标记完成状态和完成日期。
3. 有明确次数、期限或试运行目标的记录文件，完成后默认删除；只把结论沉淀到稳定文档或 ADR。
4. 删除或归档计划前，应确认 README、`docs/plans/README.md` 和相关文档链接已同步更新。
