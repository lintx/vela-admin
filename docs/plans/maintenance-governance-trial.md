# 维护规范试运行记录

本文件用于记录维护规范在真实任务中的触发情况。记录目的不是审计个人操作，而是判断规则是否清晰、是否过重、是否需要调整。

## 记录方式

每次非小改任务结束前，AI 助手应提醒维护者是否记录本次任务。只有在维护者确认后，才写入本文件。

建议记录：

1. 任务日期。
2. 任务类型。
3. 触发的规范。
4. 规则是否清晰。
5. 是否需要调整规范。

- [ ] 删除本试运行记录，或在确认有长期查阅价值时标记完成并保留。
