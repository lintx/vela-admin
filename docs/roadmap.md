# 路线图

本文记录 Vela Admin 后续维护方向，用于安排公开项目的持续迭代。

## 近期

- 完善框架包发布产物，补齐稳定的 `dist` 和类型声明输出。
- 补充 create 包交互能力，例如目标目录确认、包管理器提示和模板选项。
- 完善 README 与 docs 的 npm 发布、模板使用、主题配置说明。
- 为示例工程补充更贴近真实业务的页面拆分和权限示例。

## 中期

- 增强菜单、标签页、设置中心和主题生成器的配置能力。
- 增加更多布局组合的测试和视觉验收记录。
- 提供 TypeScript 示例模板。
- 增加 changelog 和版本发布流程。

## 长期

- 稳定公开 API，减少业务项目升级成本。
- 提供更完整的主题 token 文档。
- 增加更多后台常见场景的可选模板。
- 评估是否拆分更多可独立消费的子包。
