# 依赖治理规范

本规范用于控制依赖增长、升级风险和锁文件变更。

## 依赖分类

运行时依赖会进入用户项目安装路径，应严格控制：

1. `@varlet/ui`
2. `vue`
3. 主题和框架运行所需依赖

开发依赖只服务仓库构建、测试和开发体验，但仍会影响维护成本。

## 新增依赖

新增依赖前必须说明：

1. 使用场景。
2. 现有依赖或 Varlet 无法满足的原因。
3. 是否进入 framework 运行时依赖。
4. 许可证、维护状态、体积和安全风险。
5. 是否有轻量替代方案。

没有明确收益时，不为示例页面或临时效果新增运行时依赖。

## 升级依赖

升级依赖时应说明：

1. 升级原因。
2. 影响范围。
3. 是否包含 breaking change。
4. 已运行的验证命令。

核心依赖升级至少关注：

1. Vue。
2. Vue Router。
3. Varlet。
4. Vite。
5. Vitest。

## 锁文件

`pnpm-lock.yaml` 只应随依赖变更一起修改。仅改文档、样式或业务逻辑时，不应产生锁文件变更。

如果锁文件变化超出预期，应先说明原因并确认是否继续。

## 依赖安全

遇到安全公告或漏洞修复时，应优先评估：

1. 是否影响运行时依赖。
2. 是否影响生成器执行路径。
3. 是否存在可利用场景。
4. 升级是否引入兼容风险。

安全修复不应顺带做无关依赖升级。
