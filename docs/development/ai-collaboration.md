# AI 协作规范

## 适用范围

本文件用于约束 AI 助手在 Vela Admin 仓库内的工作方式。人类贡献者可参考，但项目根级 `AGENTS.md` 是 AI 必读入口。

## 开始任务前

AI 助手先确认：

1. 当前任务是否需要修改文件。
2. 修改是否只发生在项目目录内。
3. 是否需要读取 `docs/development/` 中的具体规范。
4. 是否需要运行命令、安装依赖、启动服务或使用浏览器。

涉及写操作时，先说明计划、影响范围、风险和验收方式，等待确认。

除只读检查、解释代码、单处错别字或链接修正等极小任务外，实际开发默认遵循 [AI 沙盒开发流程](ai-sandbox-workflow.md)。沙盒要求覆盖源码、文档、配置、Git 跟踪文件、未跟踪文件和被忽略的 `.task-plans/` 文件。用户确认方案不等于允许跳过沙盒；只有用户明确说“允许直接在主目录修改”或同等含义时才可跳过。

非极小写操作前必须完成：

1. 读取本文件、`AGENTS.md` 和本次任务相关的 `docs/development/` 规范。
2. 检查主目录 `git status --short`，确认没有未处理改动。
3. 确认本轮验收批次基点，并对齐 `.ai-dev/` 沙盒。
4. 在会话 todo/checklist 中明确标记“沙盒修改、checkpoint、patch 回主目录审核与验收”步骤。
5. 在完成上述步骤前，不得调用写文件工具、删除文件、移动文件或执行会改变主目录内容的命令。

## 计划管理

除错别字、单处链接修正、只读检查等很小修改外，AI 助手必须先制定计划，再按计划执行。

计划分两类：

1. 长期计划：影响路线图、架构、规范、发布或跨任务维护，应写入 `docs/plans/` 并加入 git 跟踪。
2. 短期计划：只服务当前任务执行，应放入 `.task-plans/`，不纳入 git 跟踪，任务结束后删除。

执行计划必须使用可更新 todo/checklist；同一时间只允许一个事项进行中，阶段完成后立即更新。会话 todo 不能替代 `.task-plans/` 或 `docs/plans/` 文件。计划文件关闭必须经维护者确认；长期计划完成后，先把结论合并到稳定文档、路线图或 ADR，再确认删除、保留或继续追踪。

## 成本控制

AI 助手应把同一对话中的上下文、已验证结果和用户确认过的设计视为可复用信息，避免把每个小反馈都当成全新任务从头执行。

1. 遇到 UI 或浏览器问题，先确认端口、工作区、页面路径、视口和 DOM，再写自动化检查。
2. 不凭记忆猜选择器；源码可读先读源码，页面状态不确定先做小范围 DOM 快照。
3. 同类浏览器验收复用脚本片段；发现端口或热更新异常时先定位来源，不反复启动服务。
4. 同一对话中已在当前基点通过的全量验证，不因纯文案或等价样式微调重复执行；说明复用依据和补充验证。
5. 连续 UI 反馈合并成一个视觉调整批次，一次修改、一次定向验证；只有触及公开 API、构建边界或跨模块逻辑才升级全量验证。

## 执行原则

1. 使用中文说明计划、进度、问题和总结。
2. 优先读取现有代码和文档，再提出方案。
3. 最小化修改，不改无关文件。
4. 不主动提交 commit，不创建分支，不改写历史。
5. 不覆盖用户已有改动。
6. 不留下无关临时文件、生成物或格式化噪音。

## UI 任务流程

涉及 UI 时，AI 助手应：

1. 先读 `docs/varlet-admin-design-system.md`。
2. 先判断 Varlet 是否已有组件。
3. 说明使用 Varlet、Vela Admin 封装、第三方包或手写 UI 的理由。
4. 完成后按 `testing-and-verification.md` 做对应验收。

## 文档任务流程

涉及文档时，先判断面向用户还是维护者，放到正确目录，更新相关入口链接，避免复制已有规则造成漂移。

## 决策沉淀

开发过程中如果出现影响长期维护的决策，AI 助手应提醒用户是否记录到 `docs/adr/`。

常见触发场景包括公开 API、运行时依赖、目录结构、发布边界、模板同步、兼容/弃用策略、UI 设计系统或组件选型原则变化。

## 结束任务前

最终回复应包含：

1. 实际修改的文件。
2. 已运行的验证命令。
3. 未能验证的部分和原因。
4. 后续建议，仅在确实有价值时提出。

如果本次任务沉淀出可复用的维护规则，应同步检查 `AGENTS.md`、`docs/development/` 和项目级 skill 是否需要更新，避免规则只留在对话记录中。影响长期维护的取舍仍应提醒维护者是否记录到 `docs/adr/` 或当前有效的长期计划。
