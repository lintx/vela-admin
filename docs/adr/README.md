# 架构决策记录

本目录用于记录影响 Vela Admin 长期维护的决策。ADR 应聚焦“为什么这样决定”，而不是重复最终代码实现。

## 何时记录

以下情况应提醒维护者是否新增 ADR：

1. 新增或改变公开 API。
2. 引入重要运行时依赖。
3. 调整目录结构、发布边界或模板同步机制。
4. 制定兼容、弃用或迁移策略。
5. 改变 UI 设计系统、主题机制或组件选型原则。

## 命名建议

```text
docs/adr/0001-title.md
```

标题使用简短英文或拼音 slug，正文使用中文。

## 内容结构

建议包含：

```md
# 决策标题

## 背景

## 决策

## 影响

## 替代方案
```
