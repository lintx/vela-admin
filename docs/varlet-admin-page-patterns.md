# 页面模式与视觉验收

本文档记录 Vela Admin 设计系统中的页面级模式、可访问性要求和视觉验收清单。组件、布局、主题 token 与导航规则见 [Varlet Admin 设计系统](varlet-admin-design-system.md)。

## 登录页

登录页默认提供框架级骨架，但业务可以完全替换。

默认方向：

- 产品信息 + 登录表单。
- 不做营销式 hero。
- 左侧只展示简洁产品信息和能力点。
- 右侧是登录表单。
- 移动端改为单列，隐藏或压缩左侧产品信息。
- 示例账号只在示例工程和开发模式显示，普通业务可关闭。

表单规则：

- 必须有真实 label，不只依赖 placeholder。
- 登录提交必须有 loading、成功、错误反馈。
- 错误使用文字和 `aria-live`，不只靠红框。
- 表单控件正式实现使用 Varlet 真实组件。

## 错误页

内置错误页必须有视觉资产，不允许只有文字和按钮。

默认规则：

- 403、404、500 默认使用布局内错误页，保留后台上下文。
- 401 或登录失效默认跳转登录页，也可配置全屏 401 页面。
- 默认插画风格为抽象 Material 插画。
- 不使用带水印、风格廉价或过时的人物插画。
- 403、404、500、网络错误分别使用不同核心图形，但共享同一插画语言。
- 图片资源必须无水印、无文字，并适配浅色和深色背景。
- 错误页包含状态码、标题、说明和恢复操作。
- 至少提供返回首页操作。
- 403 可提供重新登录或联系管理员入口。
- 业务可替换所有错误页组件，也可替换单个错误页插画资源。

## 内容区域

内容区域应服务后台效率。

默认：

- 页面背景使用 body。
- 内容区不要统一包一层大圆角卡片。
- 页面内业务卡片由业务页面自行决定。
- 框架示例页应展示真实表格、表单、搜索、详情，不做营销式 hero。
- 正式实现时业务组件应使用 Varlet 或 Vela Admin 真实组件，不使用临时占位元素替代最终控件。
- 页面内可以使用 Varlet card，但避免卡片套卡片和大面积嵌套卡片。

## 基础业务页

第一版示例工程聚焦框架核心，页面可以简单，但不能是空壳。

必须包含：

- 控制台：简单指标、最近操作、当前用户/权限/主题状态。
- 用户列表：搜索表单、工具栏、表格、分页、权限按钮。
- 用户详情：动态参数路由、基础表单、保存反馈。
- 多级菜单：至少展示到四级。
- 登录页：mock 登录。
- 403/404：验证权限和路由错误。
- 设置中心：布局、主题、侧栏配置。
- 主题生成器：第一版至少支持源颜色生成和浅/深预览。

可以后续增强：

- 复杂控制台图表。
- 完整 CRUD。
- 批量编辑真实逻辑。
- 大量业务页面。
- 复杂数据权限。
- 图片取色和开发者 token 导出可以按路线图逐步完善。

## 可访问性

必须满足：

- focus 状态可见。
- 菜单项可键盘访问。
- 菜单搜索支持键盘上下选择和回车跳转。
- 触控目标不小于 44px。
- 颜色不是唯一状态表达，active 还需要指示条。
- `prefers-reduced-motion` 下关闭或弱化动画。
- 提供 skip link，允许键盘用户跳到主内容。
- 表单输入必须有 label。
- 错误提示必须可被辅助技术感知。

## 视觉验收

每次布局或主题改动后至少检查：

- 375px 手机宽度。
- 768px 平板宽度。
- 1024px 小桌面。
- 1440px 桌面。
- 浅色 MD3。
- 深色 MD3。
- MD2 切换。
- 侧栏展开。
- 侧栏收缩。
- 多级菜单到四级。
- 长菜单滚动条。
- 收缩态浮层子菜单。
- 顶部导航。
- 命令面板搜索。
- 标签页溢出和更多菜单。
- 设置中心抽屉。
- 主题生成器预览。
- 登录页。
- 错误页插画。
- 基础业务页。

### 设计评审与高保真验收边界

设计评审用于确认设计原则、组件结构和交互方向，不直接冻结所有像素细节。

设计评审需要确认：

- 信息结构。
- 组件状态。
- 交互方式。
- 默认配置值。
- 明显影响体验的尺寸和层级。

设计评审不默认冻结：

- 图标最终形状。
- 文字精确字重。
- 阴影精确透明度。
- 面板最终宽度。
- 组件内部像素级 padding。

真实实现后必须进行高保真验收。高保真验收应基于实际 Vue + Varlet 组件、真实 CSS variables、真实图标组件和真实响应式行为，而不是静态示意图。

高保真验收发现的问题，应回写到本设计系统和对应组件实现中。

不得通过验收的表现：

- 大面积高饱和主题色。
- 纯色 active 菜单压过内容。
- 隐藏侧栏滚动条。
- 收缩态图标过小。
- 父级和当前页使用同等强度背景。
- 侧栏、顶栏和内容区域同时使用重阴影。
